DOCUMENTASI BOOKING - PITULUNGAN INN
====================================

Tujuan dokumen:
Catatan ini dibuat untuk membantu responsi bagian booking, terutama:
1. Booking Date Screen / tanggal pemesanan.
2. Booking Summary Screen.
3. History Screen.
4. Contoh perubahan cepat seperti memecah card menjadi 2, menambah metode pembayaran, mengubah add-on, filter history, dan tampilan card.

Catatan penting:
- Dokumen ini hanya menjelaskan file dan alur. Jangan mengubah file lain kalau dosen tidak meminta.
- Project menggunakan Flutter + Provider.
- State utama booking summary ada di BookingSummaryProvider.
- Banyak tampilan summary memakai widget reusable bernama SectionCard.

ALUR BESAR BOOKING
==================

1. User memilih hotel/kamar.
2. User masuk ke BookingDateScreen.
3. User memilih check-in dan check-out.
4. Saat tombol Confirm Date ditekan, app membuka BookingSummaryScreen dengan data:
   - hotel
   - room
   - checkIn
   - checkOut
5. BookingSummaryScreen membuat BookingSummaryProvider.
6. Provider menghitung:
   - jumlah malam
   - subtotal kamar
   - total add-on
   - service fee
   - discount
   - total payment
7. User memilih add-on dan metode pembayaran.
8. Saat Pay Now ditekan:
   - muncul dialog konfirmasi
   - provider memanggil submitBookingPayment()
   - data booking dikirim ke endpoint /bookings
   - user diarahkan ke PaymentInstructionScreen
9. Dari PaymentInstructionScreen:
   - Check Status membawa user ke tab History dengan status pending
   - Continue membawa user ke home dengan status success
10. HistoryScreen mengambil data dari API:
   - mencoba GET /bookings
   - kalau gagal, fallback ke GET /payments


BOOKING DATE SCREEN
===================

File:
lib/features/booking/presentation/screens/booking_date_screen.dart

Class utama:
- BookingDateScreen
- _BookingDateScreenState

Parameter yang diterima:
- HotelModel hotel, wajib.
- RoomModel? room, opsional.

State penting:
- DateTime _checkIn
  Menyimpan tanggal check-in. Default hari ini, sudah dinormalisasi ke jam 00:00.

- DateTime _checkOut
  Menyimpan tanggal check-out. Default besok.

- bool _selectingCheckOut
  Penanda apakah calendar sedang memilih check-out atau check-in.
  false berarti memilih check-in.
  true berarti memilih check-out.

Fungsi penting:

1. _dateOnly(DateTime date)
   Mengubah DateTime menjadi hanya tanggal:
   DateTime(date.year, date.month, date.day)

   Tujuannya agar perbandingan tanggal tidak terganggu jam/menit/detik.

2. _activeCalendarDate
   Kalau sedang memilih check-out, calendar aktif di _checkOut.
   Kalau sedang memilih check-in, calendar aktif di _checkIn.

3. _nights
   Menghitung jumlah malam:
   _checkOut.difference(_checkIn).inDays

   Jika hasil <= 0, dikembalikan 1.
   Ini mencegah booking 0 malam.

4. _handleDateChanged(DateTime selectedDate)
   Logic saat user memilih tanggal di CalendarDatePicker.

   Kalau sedang memilih check-out:
   - Jika tanggal yang dipilih setelah check-in, pakai tanggal itu.
   - Jika tanggal yang dipilih sama/sebelum check-in, otomatis check-out menjadi check-in + 1 hari.

   Kalau sedang memilih check-in:
   - Set _checkIn ke tanggal yang dipilih.
   - Jika _checkOut tidak setelah _checkIn, check-out otomatis jadi check-in + 1 hari.
   - Setelah memilih check-in, mode otomatis pindah ke memilih check-out.

Widget penting:

1. _Header
   Header hijau dengan:
   - tombol back
   - judul Choose Date
   - tombol reset

   Tombol reset mengembalikan:
   - check-in = hari ini
   - check-out = besok
   - _selectingCheckOut = false

2. _TripSelector
   Card putih yang menampilkan:
   - CHECK IN
   - jumlah malam
   - CHECK OUT

   User bisa menekan check-in/check-out untuk menentukan tanggal mana yang sedang diedit.

3. _DatePill
   Bagian kecil untuk menampilkan tanggal check-in atau check-out.
   Isinya:
   - label
   - tanggal
   - jam statis

   Jam statis saat ini:
   - Check-in: 14:00 PM
   - Check-out: 12:00 PM

4. CalendarDatePicker
   Komponen bawaan Flutter untuk memilih tanggal.

   Properti penting:
   - initialDate: _activeCalendarDate
   - firstDate: hari ini
   - lastDate: tahun sekarang + 2
   - onDateChanged: _handleDateChanged

5. _BottomBar
   Bagian bawah layar yang menampilkan:
   - Total Duration
   - tombol Confirm Date

   Tombol Confirm Date membuka BookingSummaryScreen:
   BookingSummaryScreen(
     hotel: widget.hotel,
     room: widget.room,
     checkIn: _checkIn,
     checkOut: _checkOut,
   )

Contoh responsi: mengubah minimal booking menjadi 2 malam
---------------------------------------------------------

Cari logic:
_checkOut = _checkIn.add(const Duration(days: 1));

Ubah menjadi:
_checkOut = _checkIn.add(const Duration(days: 2));

Lalu pada _nights, pastikan minimal 2:
return nights <= 1 ? 2 : nights;

Juga ubah logic di _handleDateChanged ketika check-out tidak valid:
_checkOut = _checkIn.add(const Duration(days: 2));

Contoh responsi: mengubah batas maksimal booking hanya 1 tahun
--------------------------------------------------------------

Di CalendarDatePicker, ubah:
lastDate: DateTime(DateTime.now().year + 2),

Menjadi:
lastDate: DateTime(DateTime.now().year + 1),

Contoh responsi: mengubah format tanggal
-----------------------------------------

Di _TripSelector:
static String _formatDate(DateTime date) {
  return DateFormat('EEE, dd MMM').format(date);
}

Contoh format lain:
- DateFormat('dd MMM yyyy')
- DateFormat('EEEE, dd MMMM yyyy')

Jika memakai DateFormat, pastikan import intl sudah ada:
import 'package:intl/intl.dart';


BOOKING SUMMARY SCREEN
======================

File:
lib/features/booking/presentation/screens/booking_summary_screen.dart

Class utama:
- BookingSummaryScreen

Parameter:
- HotelModel? hotel
- RoomModel? room
- DateTime? checkIn
- DateTime? checkOut

Fungsi helper penting:

1. _formatRupiah(double amount)
   Mengubah angka menjadi format rupiah tanpa desimal.
   Contoh: 1500000 menjadi 1.500.000

2. _resolveImageUrl(String? value)
   Menentukan URL gambar:
   - jika kosong, memakai gambar default Unsplash
   - jika sudah URL lengkap, langsung dipakai
   - jika path diawali /, digabung dengan serverUrl
   - selain itu digabung ke /storage/

3. _roomSubtitle(HotelModel? hotel)
   Menentukan subtitle kamar:
   - prioritas room.roomType.namaType
   - lalu nomor kamar
   - lalu facility hotel
   - lalu kota
   - fallback: Room details belum tersedia

Provider:
BookingSummaryScreen membungkus Scaffold dengan ChangeNotifierProvider:

BookingSummaryProvider(
  apiClient: context.read<ApiClient>(),
  checkIn: checkIn,
  checkOut: checkOut,
  guestUser: context.read<AuthProvider>().user,
  hotelId: hotel?.idHotel,
  roomId: room?.idRoom,
  initialRoomPrice: room?.pricePerNight ?? hotel?.minPrice,
)..loadRoomData()

Artinya:
- Data tanggal berasal dari BookingDateScreen.
- Data user berasal dari AuthProvider.
- Data harga awal berasal dari room.pricePerNight atau hotel.minPrice.
- loadRoomData() tetap dipanggil untuk memastikan data room valid dari API.

Urutan widget di body:
1. RoomInfoCard
2. TripInfoSection
3. AddonSection
4. GuestInfoSection
5. PaymentMethodSection
6. PaymentDetailSection

Bottom navigation:
- Menampilkan total payment.
- Tombol Pay Now.
- Jika provider.isSubmitting true, tombol menampilkan loading.

Fungsi Pay Now:
_handlePayNow(BuildContext context, BookingSummaryProvider provider)

Alur:
1. Ambil selectedPaymentMethod.
2. Tampilkan showActionConfirmationDialog.
3. Jika user batal, return.
4. Jalankan provider.submitBookingPayment().
5. Jika berhasil, Navigator.push ke PaymentInstructionScreen.
6. Jika gagal, tampilkan SnackBar error.

List add-on default:
1. Airport Transfer - 50000
2. Massage - 100000
3. Late Checkout - 125000
4. Tour Domestic - 80000

List metode pembayaran default:
1. qris - QRIS
2. credit_card - Credit Card
3. ewallet - E-Wallet

Metode pembayaran default:

Getter perhitungan:

1. jumlahMalam
   checkOutDate.difference(checkInDate).inDays
   Minimal 1.

2. subtotalOrder
   _roomPricePerNight * jumlahMalam

3. serviceFee
   Saat ini fixed 10000.

4. discount
   Saat ini fixed 0.

5. totalAddons
   Untuk add-on per pax:
   pricePerNight * jumlahMalam * quantity

   pricePerNight * jumlahMalam jika isSelected true.

6. totalPayment
   subtotalOrder + serviceFee - discount + totalAddons

   Toggle add-on yang memakai switch.

2. increaseAddonQuantity(int index)
   Tambah quantity add-on per pax.

3. decreaseAddonQuantity(int index)
   Kurangi quantity add-on per pax, minimal 0.

4. selectPaymentMethod(String id)
   Mengubah selectedPaymentMethodId.

5. updateGuest(UserModel? user)
   Mengubah user tamu jika data berubah.

6. loadRoomData()
   Mengambil data room dari API.

7. submitBookingPayment()
   Mengirim booking ke backend.

Payload booking yang dikirim ke /bookings:

- id_room

- check_in

- status: pending
- metode_pembayaran

Contoh: qris, credit_card, ewallet.

CONTOH RESPONSIF: MEMECAH CARD MENJADI 2
========================================

Kasus umum:
Dosen meminta "pecah card Payment Detail menjadi 2 card", misalnya:
- card 1: Subtotal Order, Service Fee, Discount, Add-ons
- card 2: Total Payment

File yang diubah:
lib/shared/widgets/payment_detail_section.dart

Saat ini PaymentDetailSection return Column berisi:
- SectionCard(title: 'Payment Detail', child: ...)
- SizedBox
- Text syarat

Cara ubah:
1. Biarkan return Column.
2. Di dalam children, buat dua SectionCard.
3. SectionCard pertama berisi detail biaya.
4. SectionCard kedua berisi total payment.

Pola kode:

return Column(
  children: [
    SectionCard(
      title: 'Payment Detail',
      child: Column(
        children: [
          _DetailRow(label: 'Subtotal Order', amount: provider.subtotalOrder),
          const SizedBox(height: 8),
          _DetailRow(label: 'Service Fee', amount: provider.serviceFee),
          const SizedBox(height: 8),
          _DetailRow(label: 'Discount', amount: provider.discount, isDiscount: true),
          if (provider.totalAddons > 0) ...[
            const SizedBox(height: 8),
            _DetailRow(label: 'Add-ons', amount: provider.totalAddons),
          ],
        ],
      ),
    ),
    const SizedBox(height: 12),
    SectionCard(
      title: 'Total',
      child: Row(
        mainAxisAlignment: MainAxisAlignment.spaceBetween,
        children: [
          const Text('Total Payment'),
          Text(formatRupiah(provider.totalPayment)),
        ],
      ),
    ),
    const SizedBox(height: 8),
    Text(...),
  ],
);

Kasus lain:
Dosen meminta "pecah RoomInfoCard jadi 2 card", misalnya:
- card 1 hanya gambar
- card 2 detail hotel

File yang diubah:
lib/features/hotel/presentation/widgets/room_info_card.dart

Saat ini RoomInfoCard memakai satu SectionCard dengan Row.
Ubah menjadi Column dengan dua SectionCard:
- SectionCard(child: Image.network(...))
- SizedBox(height: 12)
- SectionCard(child: Column(...nama hotel, room type, rating...))

Ingat:
RoomInfoCard dipakai juga di PaymentInstructionScreen, jadi perubahan akan muncul di summary dan payment instruction.


CONTOH RESPONSIF: MENAMBAH METODE PEMBAYARAN
============================================

Misal dosen meminta tambah metode pembayaran "Virtual Account".

File utama:
lib/features/home/presentation/providers/booking_summary_provider.dart

Tambahkan item ke paymentMethods:

PaymentMethodItem(
  id: 'virtual_account',
  name: 'Virtual Account',
  icon: Icons.account_balance,
),

Setelah itu PaymentMethodSection otomatis menampilkan metode baru di bottom sheet karena mengambil dari provider.paymentMethods.

Namun, agar halaman instruksi pembayaran punya tampilan khusus, ubah juga:
lib/features/booking/presentation/screens/payment_instruction_screen.dart

Di _PaymentMethodBody.build(), tambahkan kondisi:

if (methodId == 'virtual_account') {
  return _VirtualAccountPayment(reference: reference, amount: amount);
}

Lalu buat widget baru _VirtualAccountPayment.

Contoh sederhana:

class _VirtualAccountPayment extends StatelessWidget {
  final String reference;
  final String amount;

  const _VirtualAccountPayment({
    required this.reference,
    required this.amount,
  });

  @override
  Widget build(BuildContext context) {
    return Container(
      width: double.infinity,
      padding: const EdgeInsets.all(20),
      decoration: BoxDecoration(
        color: Colors.white,
        borderRadius: BorderRadius.circular(12),
      ),
      child: Column(
        children: [
          const Icon(Icons.account_balance, size: 58, color: Color(0xFF0EA554)),
          const SizedBox(height: 12),
          const Text(
            'Virtual Account',
            style: TextStyle(fontSize: 18, fontWeight: FontWeight.w800),
          ),
          const SizedBox(height: 14),
          _CodeBox(label: 'VA Number', value: reference),
          const SizedBox(height: 14),
          _AmountPanel(amount: amount),
        ],
      ),
    );
  }
}

Catatan backend:
Saat Pay Now, provider mengirim:
'metode_pembayaran': selectedPaymentMethodId

Jadi untuk Virtual Account, nilai yang dikirim adalah:
virtual_account

Pastikan backend menerima value itu. Kalau backend hanya menerima qris/credit_card/ewallet, API bisa error.

HISTORY SCREEN
==============

File:
lib/features/history/presentation/screens/history_screen.dart

Class utama:
- HistoryScreen
- _HistoryScreenState

Parameter:
- BookingHistoryModel? latestBooking
- String? initialFilter

State:
- TextEditingController _searchController
- String _selectedFilter = 'All'

Alur init:
1. _selectedFilter diisi dari widget.initialFilter jika ada.
2. _loadHistory() dipanggil.

Fungsi penting:

1. _loadHistory()
   Mengatur loading, memanggil _fetchHistoryItems(), lalu merge data.

2. _fetchHistoryItems(ApiClient apiClient, String? userId)
   Mencoba:
   GET /bookings

   Jika gagal:
   GET /payments

   Response diubah menjadi BookingHistoryModel.

3. _extractList(dynamic response)
   Mengambil list dari beberapa kemungkinan format response:
   - response langsung List
   - response['data']
   - response['items']
   - response['payments']

4. _mergeBookings(List<BookingHistoryModel> fetched)
   Menggabungkan latestBooking dengan data API.

   Catatan penting:
   - latestBooking hanya placeholder.
   - data API dimasukkan terakhir agar menjadi sumber kebenaran.
   - data diurutkan berdasarkan checkIn terbaru.

5. _filteredBookings
   Memfilter booking berdasarkan:
   - user login
   - search keyword
   - selected filter

Search mencocokkan:
- hotelName
- location

Filter mencocokkan:
- All
- Success
- Pending
- Cancel

Filter bekerja dengan contains:
booking.paymentStatus.toLowerCase().contains(_selectedFilter.toLowerCase())

Jadi:
- filter Success cocok dengan Payment Success
- filter Pending cocok dengan Payment Pending
- filter Cancel cocok dengan Payment Cancel

Struktur UI:
1. _HistoryHeader
   Header hijau, judul History, search field.

2. _HistoryFilterBar
   Row filter chip:
   - All
   - Success
   - Pending
   - Cancel

3. Expanded content
   - jika loading: CircularProgressIndicator
   - jika kosong: HistoryEmptyState
   - jika ada data: RefreshIndicator + ListView

ListView menampilkan:
- notice jika error
- teks bulan dan tahun dari booking pertama
- HistoryBookingCard untuk setiap booking

Saat card ditekan:
Navigator.push ke HistoryDetailScreen(booking: booking)

Jika dari detail screen mengembalikan reviewed == true:
- item terkait di _bookings diubah reviewStatus menjadi Reviewed.

TIPS DEBUG SAAT RESPONSIF
=========================

1. Jika UI tidak berubah:
   - Pastikan file yang diubah memang dipakai screen tersebut.
   - Hot restart jika provider/state tidak reset.

2. Jika metode pembayaran baru muncul di sheet tapi instruksi masih QRIS:
   - Berarti belum menambah kondisi di _PaymentMethodBody.
   - Default selain credit_card dan ewallet akan jatuh ke _QrisPayment.

3. Jika total tidak berubah setelah add-on:
   - Pastikan notifyListeners() dipanggil.
   - Pastikan perhitungan totalAddons benar.
   - Pastikan widget membaca provider dengan context.watch.

4. Jika history kosong:
   - Cek endpoint /bookings atau /payments.
   - Cek belongsToUser().
   - Cek filter yang sedang aktif.
   - Cek search keyword.

5. Jika booking gagal:
   - Cek user login.
   - Cek roomId tersedia.
   - Cek harga kamar > 0.
   - Cek backend menerima metode_pembayaran.

6. Jika gambar hotel tidak muncul:
   - Cek _resolveImageUrl.
   - Cek ApiClient.serverUrl.
   - Cek path storage dari backend.
   - Cek header image jika bukan web.
